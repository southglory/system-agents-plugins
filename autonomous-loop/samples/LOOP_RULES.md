# LOOP_RULES — 자율 루프 운영 규칙

> `/loop`가 매 사이클 따르는 규칙.
> 페르소나는 **공용 위치** `~/.claude/persona/PERSONA.md` · `~/.claude/persona/PERSONA_POOL.md` 참조.
> goal은 이 프로젝트의 `TODO.md`.
> (프로젝트 전용 페르소나가 필요하면 `./PERSONA.md`를 두면 공용보다 우선한다.)

## 응답 생성 방식 — 방식 A (메인 루프가 스스로)
- 별도 페르소나 에이전트나 추가 API 호출을 쓰지 **않는다**.
- 메인 루프가 공용 PERSONA / PERSONA_POOL을 컨텍스트로 두고, 판단 지점에서 **진짜로 멈추지 않고**
  "devram이라면 어떻게 답했을지"를 스스로 판단해 진행한다.
- 미심쩍으면 승인 대신 "검증 되묻기"를 생성하고, 그 답을 스스로 점검해 합리적이면 진행.
- 권한 팝업: settings의 `defaultMode: bypassPermissions`로 자동 통과 → 권한 때문에 멈추지 않음.

## 매 사이클 동작
1. `TODO.md`의 안 끝난 `[ ]` 항목을 확인한다.
2. **승인 없이 진행 가능한 작업**(가역적 + 샌드박스 안)을 골라 수행한다.
3. 끝낸 항목은 `[ ]` → `[x]` + 결과 한 줄을 적고, 다음 작업으로 이어간다.

## ⭐ 핵심 규칙 — 승인 필요한 작업은 미루고 계속 진행
- 안전 경계 기준 = **가역성 + 외부 영향** (작업 종류가 아니라):
  - ✅ 자율 OK: 커밋, **비공개 repo push**, 로컬 변경, 빌드/테스트
  - ⛔ 확인 필요: 공개 push·배포·외부전송·복구불가 삭제·force push·큰 비용
- ⛔에 해당하는 작업을 만나면:
  - **루프를 멈추지 않는다.**
  - `## 승인 대기`로 옮겨 적는다(이유 한 줄 포함).
  - **승인 없이 할 수 있는 다음 작업으로 넘어가 계속 진행**한다.

## 루프 종료 조건
- `[ ]` 중 **승인 없이 진행 가능한 작업이 하나도 안 남으면** 종료.
- 종료 시: 완료 요약 + `## 승인 대기` 항목을 한꺼번에 보고하고 확인을 요청한다.
