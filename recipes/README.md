# recipes/

여러 플러그인을 **조합해서 쓰는 방법**을 패키지로 제공하는 폴더.

각 레시피는 독립 플러그인이 아니라, 이미 설치된 두 개 이상의 플러그인을 엮는 **얇은 연결자**다.

## 현재 레시피

| 이름 | 전제 플러그인 | 효과 |
|---|---|---|
| [`unity-build-to-discord/`](unity-build-to-discord/) | `unity-gamedev` + `discord-huddle` | Unity 빌드 → GitHub Release → 채널에 다운로드 링크 자동 공지 |

## 예정 레시피

새 플러그인이 추가되면 관련 recipe를 여기에 더한다.

## 규약

새 레시피를 추가하는 구조와 절차는 레포 루트 [`CONTRIBUTING.md`](../CONTRIBUTING.md#recipe-추가-규약) 참조.

핵심은:

- 레시피는 **최소 두 개 이상의 플러그인** 을 엮는 경우에만 의미가 있다 (하나짜리는 해당 플러그인 자체의 문서 영역)
- 레시피는 **레시피 전용 코드가 거의 없어야** 한다 — 있다면 얇은 glue 스크립트나 조합 스킬 정도
- 두 플러그인 모두 최신 기본 규약대로 설치돼 있다고 가정한다
